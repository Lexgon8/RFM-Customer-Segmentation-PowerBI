# Dashboard

## 🎯 Objetivo

Documentar a estrutura do dashboard e as principais decisões de design e experiência do usuário adotadas na construção da solução.

---

## 📖 Visão Geral

O dashboard foi desenvolvido para transformar dados em informações estratégicas de forma clara, intuitiva e orientada à tomada de decisão.

A solução está organizada em três páginas complementares:

| Página | Objetivo |
|---------|----------|
| Dashboard Executivo | Visão consolidada dos principais indicadores |
| Segmentação | Distribuição dos clientes por segmento RFM |
| Análise de Clientes | Exploração detalhada da base utilizando filtros |

## 📐 Organização da Informação

O dashboard foi estruturado para conduzir a análise em três níveis:

1. Visão executiva por meio dos KPIs.
2. Análise dos segmentos de clientes.
3. Exploração detalhada das informações utilizando filtros e navegação entre páginas.

Essa organização torna a interpretação dos dados mais intuitiva e reduz o tempo necessário para obtenção dos principais insights.

---

## 🔍 Bastidores da Solução

A distribuição dos elementos foi planejada para minimizar o esforço cognitivo do usuário, mantendo uma hierarquia visual consistente e priorizando as informações de maior relevância para a tomada de decisão.

---

## ✅ Principais Decisões

- Layout organizado por níveis de análise.
- KPIs posicionados no topo da página.
- Filtros agrupados em uma única área.
- Navegação simplificada entre páginas.
- Identidade visual padronizada em todo o relatório.

---

## ✅ Conclusão

O dashboard foi desenvolvido para oferecer uma experiência de análise clara, intuitiva e orientada à tomada de decisão.
